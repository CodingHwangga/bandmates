# 시스템 아키텍처

## 개요

BandMate는 표준적인 **3계층 웹 아키텍처**를 따릅니다. React 기반의 프론트엔드, RESTful API를 제공하는 Node.js 백엔드, 그리고 MongoDB 데이터베이스로 구성됩니다. 클라이언트와 서버는 JSON 기반의 REST API로만 통신하며, 파일 업로드(녹음, 악보, MR 파일 등)는 멀티파트 폼 요청으로 처리되고 데이터베이스가 아닌 별도의 파일 스토리지에 저장됩니다.

---

## 아키텍처 다이어그램

```
┌─────────────────────────────────────┐
│           사용자 (브라우저)          │
└────────────────┬────────────────────┘
                 │ HTTPS
┌────────────────▼────────────────────┐
│        프론트엔드 (Next.js)          │
│  - 페이지 라우팅 (Next.js Router)   │
│  - UI 컴포넌트 (React)              │
│  - 상태 관리 (Context API)          │
│  - API 호출 (fetch / axios)         │
└────────────────┬────────────────────┘
                 │ REST API (JSON)
┌────────────────▼────────────────────┐
│     백엔드 API (Node.js / Express)  │
│  - 라우트 처리                      │
│  - 비즈니스 로직 (컨트롤러)         │
│  - JWT 인증 미들웨어                │
│  - 파일 업로드 처리 (Multer)        │
└──────────┬─────────────┬────────────┘
           │             │
┌──────────▼───┐   ┌─────▼──────────┐
│   MongoDB    │   │  파일 스토리지  │
│  (Mongoose)  │   │ (로컬/클라우드) │
└──────────────┘   └────────────────┘
```

---

## 계층별 역할 설명

### 프론트엔드 — Next.js + React

프론트엔드는 사용자와의 모든 인터랙션을 담당합니다. Next.js는 초기 페이지 로드 시 서버 사이드 렌더링(SSR)을 제공하고, 이후 페이지 이동은 클라이언트 사이드 라우팅으로 처리하여 빠른 사용자 경험을 제공합니다.

**주요 책임:**
- 페이지 렌더링: 대시보드, 밴드, 일정, 곡 목록, 자료실
- 사용자 입력 처리 및 폼 제출
- API 응답 데이터 표시 (일정, 곡, 파일 목록 등)
- 인증 상태 관리 (JWT를 HTTP-only 쿠키에 저장)

---

### 백엔드 — Node.js + Express

백엔드는 프론트엔드가 소비하는 REST API를 제공합니다. 모든 비즈니스 로직이 이 계층에서 처리되며, 보호된 라우트에는 JWT 미들웨어로 인증이 강제됩니다.

**주요 책임:**
- 클라이언트 요청의 유효성 검사 및 처리
- Mongoose 모델을 통해 MongoDB와 상호작용
- Multer를 이용한 파일 업로드 처리
- 구조화된 JSON 응답 반환
- 일관된 에러 처리 및 HTTP 상태 코드 관리

---

### 데이터베이스 — MongoDB

MongoDB는 애플리케이션의 모든 데이터를 도큐먼트 형태로 저장합니다. 각 컬렉션은 핵심 도메인 엔티티에 대응합니다.

**컬렉션 목록:**
- `users` — 계정 정보 및 프로필
- `bands` — 밴드 메타데이터 및 멤버 참조
- `practice_sessions` — 예정 및 완료된 합주 세션
- `songs` — 셋리스트 항목 및 연습 상태
- `practice_materials` — 업로드된 파일의 메타데이터

---

### 파일 스토리지

업로드된 파일(녹음, 악보, MR 파일)은 데이터베이스 외부에 저장됩니다. 개발 환경에서는 서버의 로컬 `/uploads` 디렉토리에 저장하며, 운영 환경에서는 클라우드 오브젝트 스토리지(예: AWS S3)로 교체하는 것을 목표로 합니다.

---

## 요청 흐름 예시

**연습 녹음 파일 업로드 흐름:**

1. 사용자가 자료 업로드 페이지에서 파일 선택
2. 프론트엔드가 파일과 메타데이터를 담아 `POST /api/materials` 멀티파트 요청 전송
3. Express 미들웨어(Multer)가 파일을 스토리지에 저장하고 파일 경로를 `req.file`에 첨부
4. 컨트롤러가 파일 경로와 메타데이터를 포함한 `PracticeMaterial` 도큐먼트를 MongoDB에 생성
5. `201 Created` 응답을 프론트엔드에 반환
6. 프론트엔드가 UI를 업데이트하여 새로 업로드된 자료를 표시

---

## 인증 흐름

BandMate는 **JWT(JSON Web Token)** 기반 인증을 사용합니다.

```
사용자가 로그인 폼 제출
        │
        ▼
POST /api/auth/login
        │
        ▼
서버가 자격 증명 확인 → JWT 발급
        │
        ▼
JWT를 HTTP-only 쿠키에 저장
        │
        ▼
이후 요청마다 JWT 포함 → 미들웨어가 토큰 검증
        │
        ▼
인증 성공 시 보호된 라우트 접근 허용 / 실패 시 401 반환
```
