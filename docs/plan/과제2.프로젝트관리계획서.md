## 1. 프로젝트 개요

### 1.1 프로젝트 목적
BandMate는 대학 밴드 동아리나 소규모 밴드 팀이 합주 일정, 연습곡, 연습 자료를 효율적으로 관리할 수 있는 웹 기반 통합 플랫폼을 개발하는 것을 목표로 한다.

### 1.2 프로젝트 배경
현재 밴드 팀들은 다음과 같은 문제를 겪고 있다:
- 합주 일정이 카카오톡/메신저에 흩어져 관리됨
- 연습곡 리스트와 악보, MR 파일이 체계적으로 관리되지 않음
- 파트별 연습 자료 공유가 어려움
- 합주 녹음이나 연습 기록이 남지 않음

이러한 문제를 해결하기 위한 통합 관리 플랫폼이 필요하다.

### 1.3 프로젝트 범위

**포함 사항 (In Scope)**:
- 밴드 생성 및 팀 관리
- 합주 일정 관리 (캘린더 형태)
- 연습곡 리스트 관리 및 세트리스트 구성
- 파트별 연습 자료 공유 (기타 TAB, 베이스 TAB, 드럼 악보, MR 파일)
- 합주 기록 저장 (녹음 파일, 메모)

**제외 사항 (Out of Scope)**:
- 실시간 영상 통화 기능
- 음원 스트리밍 서비스
- 금전 관리 (회비, 수익 배분)
- 모바일 네이티브 앱 (웹 기반만 제공)

## 7. 형상 관리

### 7.1 형상 관리 방침
본 프로젝트는 별도로 작성된 **형상관리 계획서**를 따른다.

주요 내용:
- Git을 활용한 버전 관리
- 형상 항목: 요구사항서, 설계서, 계획서, 테스트 문서, 소스코드
- 버전 규칙: v주.부.수 (Semantic Versioning)
- **주 1회 이상 commit 필수**

### 7.2 브랜치 전략

| 브랜치 | 용도 |
|--------|------|
| **main** | 최종 승인된 산출물 |
| **draft** | 작업 중인 문서/코드 |
| **review** | 검토 대기 중 |
| **experiment** | 실험용 (삭제 가능) |

### 7.3 Commit 메시지 규칙

```
[구분자] 변경 내용

예시:
[FEAT] 밴드 생성 API 구현
[FIX] 일정 수정 버그 해결
[DOCS] 요구사항 정의서 작성
[CR-01] 파일 업로드 용량 제한 추가
```

### 7.4 주요 태그

| 태그명 | 시점 |
|--------|------|
| requirements_freeze | 요구사항 확정 (4/27) |
| design_complete | 설계 완료 (5/11) |
| beta_release | 베타 버전 (6/8) |

## 8. 변경 관리

### 8.1 변경 요청 프로세스

```
1. 변경 필요성 인지
   ↓
2. 변경요청서(CR) 작성
   - CR 번호 부여 (CR-01, CR-02, ...)
   - 변경 사유 기록
   ↓
3. 영향도 분석
   - 일정 영향
   - 기술적 영향
   - 문서 변경 범위
   ↓
4. 승인 여부 결정
   - 자체 판단 또는 교수 자문
   ↓
5. 변경 구현
   - 별도 브랜치 생성
   - 코드/문서 수정
   ↓
6. 테스트 및 검증
   ↓
7. main 브랜치 병합
   ↓
8. 변경 이력 기록 (CHANGELOG.md)
```

### 8.2 변경 통제 위원회 (CCB)
- 1인 프로젝트로 자체 판단
- 중요 변경 사항은 교수 자문
